# Как мыслить при решении `aclean`

## Шаг 1: что значит «одинаковые» слова?

Не «похожие», а **анаграммы** — из одних и тех же букв в одном количестве.

`nap` = `PAN`, `teachers` = `cheaters` = `hectares`, `ear` = `era`.

Попарно сравнивать все со всеми — долго. Нужен **один признак группы**.

---

## Шаг 2: канонический ключ

Любое слово сводим к **одной строке-сигнатуре**, одинаковой у всех анаграмм:

> lowercase → все буквы → отсортировать → склеить

`nap` → `"anp"`  
`PAN` → `"anp"`  
`teachers` → `"aceehrst"` (с двумя `e`!)

Это и есть **ключ группы**.

---

## Шаг 3: какая структура данных?

Нужно: «для каждой группы хранить **одно** слово».

| Структура | Подходит? |
|-----------|-----------|
| `Set` строк | только ключи, без «представителя» слова |
| `Map` ключ → слово | да: ключ = сигнатура, значение = первое слово |

Одна `Map` на весь алгоритм.

---

## Шаг 4: один проход

Для каждого `item`:

1. посчитать сигнатуру;
2. если сигнатура **уже в Map** → `continue` (группа есть);
3. иначе → `Map.set(сигнатура, item)`.

В конце — **все значения Map** (по одному слову на группу).

---

## Схема в голове

```
вход:  nap, teachers, cheaters, PAN, ear, era, hectares

nap      → anp        → Map: { anp: "nap" }
teachers → aceehrst   → Map: { ..., aceehrst: "teachers" }
cheaters → aceehrst   → уже есть → skip
PAN      → anp        → уже есть → skip
ear      → aer        → Map: { ..., aer: "ear" }
era      → aer        → skip
hectares → aceehrst   → skip

результат: ["nap", "teachers", "ear"]
```

---

# Твои два варианта

**`aclean1`** — без `has` / `continue`: каждый раз `set` → остаётся **последний** из группы.

**`aclean`** — с `has` + `continue` → **первый** из группы. Для классического `aclean` это то, что нужно.

---

# Где ещё упростить

### 1. `letterSet` можно убрать

После `sort()` кладёшь буквы в `Set` — **теряется количество** (`teachers` и гипотетическое слово с теми же буквами, но другими повторами могут слиться неверно).

Для анаграмм достаточно: **отсортировать все буквы и join** — без `Set`.

### 2. Второй цикл и `res` не нужны

`Map` уже хранит итог. Вместо цикла с `i[1]`:

`return [...wordMap.values()]`

### 3. `forEach` внутри `for` — можно одной цепочкой

Сигнатура — одно выражение: lowercase → split → sort → join.

# Минимальная логика (без лишних деталей)

```
Map пустая
для каждого слова:
  key = нормализованное слово
  если key уже в Map → пропуск
  иначе Map[key] = слово
вернуть все значения Map
```

Три идеи: **нормализация**, **Map как группировка**, **не перезаписывать** (первый представитель).

---

# Чеклист перед «готово»

1. Ключ — **все** буквы с повторами, не уникальные.  
2. В Map пишем только если ключа ещё нет.  
3. Одна структура, один проход по `arr`, без отдельного `res`.

Попробуй переписать `aclean` в ~10 строк: один `Map`, один `for`, `continue`, `values()`. Если пришлёшь версию — разберём построчно.
